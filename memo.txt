GitHub는 2021년 8월 13일부터 HTTPS URL을 이용한 클론 및 푸시 작업 시 암호 인증을 지원하지 않습니다. 대신, GitHub는 퍼스널 액세스 토큰(PAT)이나 SSH 키를 사용한 인증을 권장합니다. 이를 해결하기 위해 다음 단계들을 따라주세요.

1. 퍼스널 액세스 토큰(PAT) 사용
퍼스널 액세스 토큰을 생성하고 이를 사용하여 인증을 할 수 있습니다.

퍼스널 액세스 토큰 생성
GitHub의 Settings 페이지로 이동합니다.
"Generate new token" 버튼을 클릭합니다.
필요한 권한 범위를 설정합니다. (예: repo, user 등)
"Generate token" 버튼을 클릭하면 토큰이 생성됩니다. 이 토큰은 한 번만 표시되므로 안전한 곳에 저장하세요.
퍼스널 액세스 토큰 사용
생성된 토큰을 사용하여 리포지토리를 클론합니다.

bash
코드 복사
git clone https://<your_username>:<your_token>@github.com/recon-reco/django_blog.git
2. SSH 키 사용
SSH 키를 생성하고 이를 GitHub에 추가하여 인증할 수 있습니다.

SSH 키 생성
로컬 시스템에서 SSH 키를 생성합니다.

bash
코드 복사
ssh-keygen -t ed25519 -C "your_email@example.com"
프롬프트에 따라 키의 경로와 암호를 입력합니다. 기본 경로를 사용하는 것이 일반적입니다.

SSH 키를 GitHub에 추가
생성된 공개 키(~/.ssh/id_ed25519.pub)를 복사합니다.
GitHub의 SSH and GPG keys 페이지로 이동합니다.
"New SSH key" 버튼을 클릭하고 키를 추가합니다.
SSH를 사용하여 리포지토리를 클론
SSH URL을 사용하여 리포지토리를 클론합니다.

bash
코드 복사
git clone git@github.com:recon-reco/django_blog.git
위 방법 중 하나를 사용하여 GitHub 리포지토리를 성공적으로 클론할 수 있습니다. 문제가 계속되면 추가적인 정보나 도움이 필요할 수 있습니다.
